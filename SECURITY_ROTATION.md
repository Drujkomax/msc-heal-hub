# Ротация ключей безопасности / Security Keys Rotation

## Обзор
Этот документ описывает процесс ротации секретных ключей для проектов `msc-heal-hub-stage` и `msc-heal-hub-prod`.

---

## Критические секреты

### 1. Supabase Anon Key & Service Role Key
**Когда ротировать:** Каждые 90 дней или при подозрении на компрометацию

**Процедура:**
1. Открыть [Supabase Dashboard](https://supabase.com/dashboard)
2. Выбрать проект (stage или prod)
3. Перейти в Settings → API
4. Сгенерировать новые ключи (кнопка "Regenerate")
5. Обновить ключи в Edge Function Secrets:
   - Settings → Edge Functions → Secrets
   - Обновить `SUPABASE_SERVICE_ROLE_KEY` (если используется)
6. **Для Lovable:** ключи обновляются автоматически в `src/integrations/supabase/client.ts`
7. Проверить работу приложения на health-странице
8. Дождаться применения изменений (~2-5 минут)
9. Ревокнуть старые ключи

**Ответственный:** Director/Admin

---

### 2. Telegram Bot Token
**Когда ротировать:** При компрометации или смене бота

**Процедура:**
1. Связаться с [@BotFather](https://t.me/BotFather) в Telegram
2. Использовать команду `/revoke` для текущего бота
3. Создать новый токен (или получить новый для существующего бота)
4. Обновить в Supabase Secrets:
   - Dashboard → Project Settings → Edge Functions → Secrets
   - Обновить значение `TELEGRAM_BOT_TOKEN`
5. Перезапустить Edge Functions (автоматически при следующем деплое)
6. Протестировать бота: отправить `/start` в Telegram

**Ответственный:** Director/Tech Lead

---

### 3. BOT_BACKEND_JWT
**Когда ротировать:** Каждые 180 дней или при подозрении на утечку

**Процедура:**
1. Сгенерировать новый JWT токен:
   ```bash
   # Пример генерации (используйте надежный генератор)
   openssl rand -base64 64
   ```
2. Обновить в Supabase Secrets (stage и prod):
   - Dashboard → Project Settings → Edge Functions → Secrets
   - Обновить `BOT_BACKEND_JWT`
3. Обновить токен в конфигурации Telegram-бота (Кодекс)
4. Протестировать взаимодействие бота с Edge Functions
5. Убедиться, что старый токен больше не используется

**Ответственный:** Director/Tech Lead

---

## Процедура экстренной ротации

### При подозрении на компрометацию:

1. **Немедленно:**
   - Ревокнуть скомпрометированный ключ
   - Проверить логи доступа (Supabase Dashboard → Logs)
   - Уведомить команду

2. **В течение 1 часа:**
   - Сгенерировать новые ключи
   - Обновить все зависимые сервисы
   - Проверить отсутствие несанкционированного доступа

3. **В течение 24 часов:**
   - Провести аудит безопасности
   - Обновить документацию
   - Создать отчет об инциденте

# Система настраиваемых прав доступа

## Описание
Директор может создавать сотрудников с гибкими настраиваемыми правами доступа к различным разделам админ панели.

## Возможности

### 1. Настраиваемые права доступа
При редактировании сотрудника доступны две вкладки:
- **Основная информация** - email, пароль, роль
- **Настройка прав** - настройка прав доступа к разделам

### 2. Уровни доступа
Для каждого раздела можно установить один из трех уровней:
- **Полный доступ** - сотрудник может просматривать и редактировать данные
- **Только просмотр** - сотрудник может только просматривать данные
- **Нет доступа** - сотрудник не видит раздел (по умолчанию)

### 3. Временные сотрудники
При настройке прав можно:
- Включить режим "Временный сотрудник"
- Установить дату истечения доступа
- После истечения срока доступ автоматически деактивируется

### 4. Доступные разделы
- Панель управления (Dashboard)
- Лиды (Leads)
- Сделки (Deals)
- Задачи (Tasks)
- Товары (Products)
- Услуги (Services)
- Контакты (Contacts)
- Категории (Categories)
- Сотрудники (Users)
- Аналитика (Analytics)
- Архив (Archive)

## Как использовать

### Создание сотрудника с настраиваемыми правами

1. Перейдите в раздел "Сотрудники"
2. Нажмите "Добавить сотрудника"
3. Заполните email и выберите базовую роль
4. После создания нажмите кнопку "Редактировать" на карточке сотрудника
5. Перейдите на вкладку "Настройка прав"
6. Выберите разделы и уровни доступа:
   - Нажмите на бейдж "Полный доступ" для полного доступа
   - Нажмите на бейдж "Только просмотр" для режима просмотра
7. При необходимости включите "Временный сотрудник" и выберите дату
8. Нажмите "Сохранить настройки"

### Временный сотрудник

1. При редактировании сотрудника перейдите на вкладку "Настройка прав"
2. Включите переключатель "Временный сотрудник"
3. Выберите дату истечения доступа в календаре
4. Настройте права доступа
5. Сохраните изменения

После указанной даты:
- Сотрудник потеряет доступ к системе
- Все настроенные права станут неактивными
- Директор может вручную деактивировать временного сотрудника до истечения срока

## Приоритет прав

Система прав работает по следующему принципу:

1. **Если настроены кастомные права** - используются только они, базовая роль игнорируется
2. **Если кастомные права не настроены** - используются стандартные права базовой роли
3. **Директор и Sales Manager** - всегда имеют полный доступ, кастомные права на них не действуют

## Технические детали

### База данных
- `employee_custom_permissions` - таблица кастомных прав доступа
- `temporary_employees` - таблица временных сотрудников

### Функции
- `has_custom_permission()` - проверка кастомных прав
- `is_temporary_employee_active()` - проверка активности временного доступа

### Хуки
- `useCustomPermissions` - управление кастомными правами
- `useUserPermissions` - проверка прав доступа (обновлен для поддержки кастомных прав)

### Компоненты
- `CustomPermissionsForm` - форма настройки прав доступа
- `EditEmployeeModal` - обновлен для поддержки вкладки настройки прав
